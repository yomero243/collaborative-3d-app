# Contribuyendo a Air Hockey 3D

¡Gracias por tu interés en contribuir a Air Hockey 3D! Queremos hacer que contribuir a este proyecto sea lo más fácil y transparente posible, ya sea:

- Reportando un error (bug)
- Discutiendo el estado actual del código
- Enviando una solución (fix)
- Proponiendo nuevas características
- Convirtiéndote en mantenedor

## Proceso de Desarrollo

Usamos Github para alojar código, rastrear problemas y solicitudes de funciones, así como aceptar Pull Requests.

1.  **Fork** el repositorio y crea tu rama desde `main`.
2.  Si has añadido código que debe ser probado, añade pruebas.
3.  Si has cambiado APIs, actualiza la documentación.
4.  Asegúrate de que el test suite pase.
5.  Asegúrate de que tu código siga el estilo existente (linter).
6.  Envía un Pull Request.

## Reportando Bugs

Usamos GitHub issues para rastrear errores públicos. Reporta un error [abriendo un nuevo issue](); es genial si incluyes:

-   Un título claro y descriptivo.
-   Pasos para reproducir el problema.
-   Lo que esperabas que sucediera vs lo que realmente sucedió.
-   Screenshots o videos si es relevante.
